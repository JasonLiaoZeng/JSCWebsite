Definitions:

Higher-Level Language

focuses more on the programming logic rather than the underlying hardware components. Will be compile down to assembly languages. (Java, Python, C++, C#, etc)

Assembly Language

bridge between the underlying hardware of a computer and the higher-level programming languages. Takes a higher-level language's logic and communicate directly with a computerâ€™s hardware. Will be converted into executable machine language. (x86, ARM, MIPS, etc)

Machine Language

binary digits of fixed length that hardware like processor can directly understand. Make up of 0s and 1s. Depending on the type of operation and hardwares, the length could be different. (0100001011110111, 0011, 01001000, etc)

Compiler

a special computer program that translates computer code written in one programming language (the source language) into another language (the target language). It's most important usage is to convert a higher-level language such as C# into executable machine code.

Object-Oriented Programing Language

a high-level computer programming language that implements objects and their associated procedures within the programming context to create software programs. (It is ok if student don't know what this means and hwo this is significant after reading the lecture page, it will be explained later with examples)

Primitive Data Type

the set of the basic data types from which all other data types are constructed.

Readings:

Upon reading the definition section, student should understand how a higher level programming language like C# turns into software. Through compiler, a higher-level language will be compiled into a lower-level language, then eventually the assembly language matching their specific harware, then assembly language gets to be turned into machine codes that's executable by their processors.

In modern days, most codes are written with higher-level languages beacause it is more productive, easier to debug, and works well across different hardwares.

Another feature of many higher level languages to help developement is to be object-oriented. As defined, In an object-oreiented language, coder must explicitly let the compiler know what type of object is attached to a variable name.

To give a brief example on one of the advantages of object oriented languages, imagine we have 2 objects, a dog object that can bark and a cat object that can meow. We now assign the dog object to a variable named Dori and cat object to a variable name Cathy. In an object-oriented language, we let the compiler know that Dori is a dog and Cathy is a cat, so it's clear to the compiler that Dori must be able to bark and Cathy must be able to meow. But, in a non-object-oriented language, the compiler won't know that Dori is a dog and Cathy is a cat, so it doesn't know what kind of option they can perform. Such information could be useful when we are developing and debugging.

// (ObjectOriented) C#

Dog Dori = new Dog();

Cat Cathy = new Cat();

// (Non-ObjectOriented) Python

Dori = Dog()

Cathy = Cat()
