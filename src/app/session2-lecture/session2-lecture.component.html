Definitions:

Higher-Level Language

focuses more on the programming logic rather than the underlying hardware components. Will be compile down to assembly languages. (Java, Python, C++, C#, etc)

Assembly Language

bridge between the underlying hardware of a computer and the higher-level programming languages. Takes a higher-level language's logic and communicate directly with a computer’s hardware. Will be converted into executable machine language. (x86, ARM, MIPS, etc)

Machine Language

binary digits of fixed length that hardware like processor can directly understand. Make up of 0s and 1s. Depending on the type of operation and hardwares, the length could be different. (0100001011110111, 0011, 01001000, etc)

Compiler

a special computer program that translates computer code written in one programming language (the source language) into another language (the target language). It's most important usage is to convert a higher-level language such as C# into executable machine code.

Object-Oriented Programing Language

a high-level computer programming language that implements objects and their associated procedures within the programming context to create software programs. (It is ok if student don't know what this means and how this is significant after reading the lecture page, it will be explained later with examples)

Primitive Data Type

the set of the basic data types from which all other data types are constructed.

Computer Architects and Programming Languages:

Upon reading the definition section, student should understand how a higher level programming language like C# turns into executable software. Through compiler, a higher-level language will be compiled into a lower-level language, then eventually the assembly language matching their specific harware, then assembly language gets to be turned into machine codes that's executable by their processors.

In modern days, most codes are written with higher-level languages beacause it is more productive, easier to debug, and works well across different hardwares.

Another feature of many higher level languages to help developement is to be object-oriented. As defined, In an object-oreiented language, coder must explicitly let the compiler know what type of object is attached to a variable name.

To give a brief example on one of the advantages of object oriented languages, imagine we have 2 objects, a dog object that can bark and a cat object that can meow. We now assign the dog object to a variable named Dori and cat object to a variable name Cathy. In an object-oriented language, we let the compiler know that Dori is a dog and Cathy is a cat, so it's clear to the compiler that Dori must be able to bark and Cathy must be able to meow. But, in a non-object-oriented language, we won't tell the compiler that Dori is a dog and Cathy is a cat, so it doesn't know what kind of action they can perform. Such information could be useful when we are developing and debugging.

// (ObjectOriented) C#

Dog Dori = new Dog();

Cat Cathy = new Cat();

// (Non-Object-Oriented) Python

Dori = Dog()

Cathy = Cat()

Again, C# is higher-level object-oriented language. Before explaining how objects are assigned to variables, we must know the primitive data types. On the other hand, arrays are defined as sequence of objects. In C# and other object-oriented languages, elements in an array must be the same type or the children type.(the concept of parent type and children type will be covered later in session 2) Moreover, another important restriction of array is that the size of the array must be known upon declaration. In another word, coders cannot change the size of array, this restriction is for memory protection. Below is some example of how to declare and inititaliz an array.

//declaring an array and initialize it at the same time

int[] a0 = { 0, 1, 3};

//declaring an array without initializing

int[] a1 = new int[3];

//declaring an array and initializing later

int[] a2 = new int[3];

a2 = { 0, 1, 3};

//another way of declaring an array and initializing later

int[] a3 = new int[3];

a3[0] = 0;

a3[1] = 1;

a3[2] = 3;

//declaring an array then partly initialize it

int[] a4 = new int[3];

a4[0] = 0;

Also be noted that in C# and many other programming languages, coders can choose their own variable name, but there are some restrictions. Here are the rules for choosing variable names:

• The name can contain letters, digits, and the underscore character (_).

• The first character of the name must be a letter. The underscore is also a legal first character, but its use is not recommended at the beginning of a name. An underscore is often used with special commands, and it's sometimes hard to read.

• Case matters (that is, upper- and lowercase letters). C# is case-sensitive; thus, the names count and Count refer to two different variables.

• C# keywords can't be used as variable names. Recall that a keyword is a word that is part of the C# language. (A complete list of the C# keywords can be found in Appendix B, "C# Keywords.")

On top of these hard restriction, it is also important to chose meaningful names and follow some kind of conventions. Although the compiler would not complain, having organized and readable code will help both the developer and future code readers.

C# - If-Statement:

If-statement is a decision-making statement that guides a program to make decisions based on specified criteria, it allows more than one possible flow of control. The key words that students should know are 'if', 'if else', and 'else', and here are some examples of if-statement in C#:

//Perform action0 function that takes in an empty parameter if condition0 is true

if (condition0) {

 action0();

}

//Perform action0 function if condition0 is true

//else perform action1 function if condition1 is true

//else perform defaultAction

if (condition0) {

 action0();

} else if (condition1) {

 action1();

} else {

 defaultAction();

}

Noted that the condtion in a if-statement should be a bool value, it could be a bool variable, bool function, or bool operation. Below are some example of if-statements those conditions are bool operations.

//For the following examples, assume variables 'a' and 'b' are int

//Perform action function if 'a' is greater than 'b'

if (a > b) {

 action();

}

//Perform action function if 'a' is less than or equal to 'b'

if (a <= b) {

 action();

}

//Perform action function if 'a - b' equals to '10'

if (a - b == 10) {

 action();

}

//Perform action function if 'a' is not equal to 'b'

if (a != 0) {

 action();

}

//Perform action function if 'a' is an even number

if (a % 2 == 0) {

 action();

}

//Perform action function if both 'a' and 'b' are 0

if (a == 0 && b == 0) {

 action();

}

//Perform action function if at least 1 of 'a' and 'b' is 0

if (a == 0 || b == 0) {

 action();

}

C# - For-loop and While-loop

Loops are use to execute statements repeatly while certain crateria is true. For-loop and While-loop are 2 types of loops that student should understand by this course. The 2 types of loops are similar and can often be replaced with the other, and the concept of While-loop is simpler of the two, it execute statement repeatedly while a statement is ture, and here is an example of While-loop:

//Print "Hello World" to concole 3 times

int i = 0;

while( i < 3) {

 Console.WriteLine("Hello World");

 i++;

}

For-loops are are similar to While-loop except that we must provide an initialization in the loop statements. Below is an example and some explanation:

//Print "Hello World" to concole 3 times

for (int i = 0; i < 3; i++) {

 Console.WriteLine("Hello World");

}

What logically happened in the above for-loop is that:

1. 'i' is initialized to 0. 0 is less than 3, so print "Hello World" to console.

2. 'i' becomes 1 due to loop statement 'i++'. 1 is less than 3, so print "Hello World" to console.

3. 'i' becomes 2 due to loop statement 'i++'. 2 is less than 3, so print "Hello World" to console.

4. 'i' becomes 3 due to loop statement 'i++'. 3 is not less than 3, so the loop ends.

Something that student should think about is that infinite loops could occur, and this will most likely cause a runtime error depending on the compiler. Below is an example of an infinite loop:

//Print "Hello World" to concole infinite times

int i = 0;

while( true) {

 Console.WriteLine("Hello World");

}

C# - functions

A function is simply a “chunk” of code that programmers can use over and over again, rather than writing it out multiple times. Functions enable programmers to break down or decompose a problem into smaller chunks, each of which performs a particular task. Once a function is created, the details of how it works can almost be forgotten about. In this way the detail is abstracted, allowing the programmer to focus on the bigger picture.
